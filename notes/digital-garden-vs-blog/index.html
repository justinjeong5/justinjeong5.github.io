디지털 가든은 왜 블로그보다 운영 부담이 적은가

시간순 블로그는 "이번 달은 안 썼네"가 누적되지만, 디지털 가든은 노트의 성숙도만 다듬어도 사이트가 살아있게 됩니다.

  • #garden
  • #writing
  • #ops

핵심 차이

블로그디지털 가든
시간축게시일 강조성숙도(🌱🌿🌳) 강조
운영 부담"마감 압박" 누적다듬기만 해도 ok
탐색시간순/카테고리토픽·백링크로 비선형
완성도"완성"이 전제미완성도 공개

왜 부담이 적은가

블로그는 "글 = 완성된 결과물". 그래서 한 편의 글을 게시하려면 도입·전개·결론을 다 갖춰야 합니다. 디지털 가든은 "글 = 자라는 노트". 한 줄 메모로 시작해 🌿로 자라고, 충분히 다듬으면 🌳가 됩니다.

이 차이가 운영에 미치는 효과는 비선형입니다.

  • 시작 비용 ↓: 완성 압박이 없어 한 줄로 시작 가능. 시작 비용이 0에 가까우면 시작 빈도가 올라감.
  • 재방문 자연스러움: 같은 글을 다시 다듬는 게 가든의 핵심 동작. 블로그는 다듬으면 "수정 이력" 같은 노이즈가 됨.
  • 상태별 분류: 미완성도 공개. 🌱이라는 신호 자체가 "이건 자라는 중"이라는 자기 인정.
  • 누적이 자라남으로 보임: 블로그는 "글 N편 쓴 사이트"로 보이고, 가든은 "N편이 서로 연결된 사이트"로 보임. 후자가 깊이 신호.

블로그가 시간순 강조라면, 가든은 연결 강조입니다. 같은 글 50편이어도 시간순 나열보다 연결된 그래프가 훨씬 깊은 신호.

운영 룰 (현재)

  • 🌱 새싹: 핵심 주장 1줄 + 근거 1-2개. 5분 분량.
  • 🌿 자라는 중: 반례·예외·옆 주제와 연결 추가. 10-20분 분량.
  • 🌳 상록수: 여러 번 다듬어 정착한 노트. 다른 노트에서 자주 백링크 받음.

함정

  • 너무 많은 🌱을 심으면 "정원이 황무지"처럼 보일 수 있음 → 🌿 이상만 인덱스 상단에.
  • 백링크가 없으면 "그래프"의 의미가 약함 → 토픽 태그 일관성이 핵심.
  • 🌱을 영원히 🌱로 두면 가든이 아니라 메모장. lastTendedAt 다듬기가 한 달 이상 없으면 노트를 삭제하거나 합치는 것도 운영.
  • 게시일 숨김이 "투명성 부족"으로 읽힐 수도. plantedAt/lastTendedAt 둘 다 frontmatter에 두고 화면에 lastTendedAt만 보이게 하면 균형.

관련 노트

  • 🌳 Incrementally correct — 한 번에 완성하지 않는 사이트

    Brian Lovin이 제안한 개념. 글이 "출간"되는 게 아니라 시간을 두고 점진적으로 정확해지는 운영 방식.

  • 🌿 옛 레포 archive할까 keep할까 — 정리 기준

    6년 누적된 30+ 학습 레포. 모두 두면 신호 노이즈, 다 지우면 흔적 0. 4가지 분류 기준으로 archive/keep/private을 결정한다.

  • 🌱 PR 본문은 코드보다 먼저 읽힌다

    리뷰어는 코드를 보기 전에 PR 본문을 본다. 본문이 비어있거나 "WIP"이면 리뷰어는 코드를 추측해서 읽는다. 본문 5분 투자로 리뷰 30분 절약.

  • 🌱 Note 라운드 패턴 — 신규 → cross-link → 함정 → 다음 라운드

    노트 50편을 모은 게 한 번에 일어난 게 아니다. 신규 양산 → 누적 후 cross-link → 함정 섹션 보강 → 다음 라운드의 4단계 사이클이 반복됐다.

  • 🌱 리빌드 vs 점진 개선 — 사이트는 리빌드, 콘텐츠는 점진

    같은 손으로 같은 사이트를 리빌드는 1일에, 콘텐츠는 3일에 걸쳐 점진으로 추가했다. 두 모드를 의도적으로 분리한 이유와 언제 어느 쪽을 고르는지.
