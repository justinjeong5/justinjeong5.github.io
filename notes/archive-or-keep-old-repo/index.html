옛 레포 archive할까 keep할까 — 정리 기준

6년 누적된 30+ 학습 레포. 모두 두면 신호 노이즈, 다 지우면 흔적 0. 4가지 분류 기준으로 archive/keep/private을 결정한다.

문제 — 30+ 학습 레포의 운영

GitHub 프로필에 6년치 학습 레포가 쌓인다.

  • 2020년 React 첫 학습 (react-tetris)
  • 2021년 백엔드 입문 (mern-stack-boilerplate)
  • 2022년 GraphQL 시도 (graphql-apollo)
  • 2023년 Webpack Module Federation (mfp)
  • 2024년 첫 회사 (j-chat)
  • ...

전부 두면:

  • 프로필이 "학습 흔적"으로 뒤덮임 (긍정 신호)
  • 하지만 "최근 작업"이 묻힘 (부정 신호)
  • 6년 전 코드 수준이 외부에 노출 (부정 신호)

전부 지우면:

  • 학습 흔적 0 → 어떻게 자라왔는지 안 보임
  • 학습 자산 0 → 6년 전 본인이 한 게 사라짐

중간이 답. 4가지 기준으로 분류.

4가지 분류

1. Keep (public, top 위치)

  • 1년 안에 만진 적 있음
  • README + license + 동작 가능
  • 본인 capability 신호로 작용
  • 다른 글·노트에서 인용 가능

→ 그대로 두기. 정기 lastTendedAt 갱신.

2. Archive (read-only)

  • 6개월+ 안 만짐
  • 학습 가치는 있는데 현재 가치 약함
  • README는 있고 동작은 함
  • "이 시점의 내가 무엇을 만들었는가" 흔적

→ GitHub archive. 검색은 되지만 push·issue 닫힘. 명시적 "더 이상 활성 아님" 신호.

3. Private (숨김)

  • 코드 수준이 부끄러움
  • 동작 안 함 또는 README 없음
  • 본인만 의미 (학습 메모)

→ GitHub private 전환. 본인은 접근, 외부엔 안 보임. 삭제보다 안전 (히스토리 보존).

4. Delete (드물게)

  • 민감 정보 포함 (.env 노출 등)
  • 1차 시도 후 즉시 다시 만든 미완성 repo
  • 학습 가치 0이고 흔적 없어도 됨

→ gh repo delete. 영구 삭제. 신중히.

분류 작업 흐름

# public 레포 목록 (push 날짜순)
gh repo list <user> --visibility public --limit 100 \
  --json name,pushedAt,description,isArchived \
  --jq 'sort_by(.pushedAt) | reverse'

# 각 repo 검토
gh repo view <user>/<repo>

# archive
gh repo archive <user>/<repo>

# private 전환
gh repo edit <user>/<repo> --visibility private

분기 1회 (3개월마다) 1시간 작업. 다음 한 분기 동안 유지.

어떤 게 keep할 가치 있나

신호keep?
회사에서 인용 가능 (RFC·온보딩)✅ keep
본인 사이트에서 cross-link✅ keep
다른 사람이 fork·star 함✅ keep (외부 가치 신호)
채용 인터뷰 시 "이거 봤어요"✅ keep
1년 동안 본인 외 0명 방문⚠ archive 후보
코드가 본인이 봐도 부끄러움⚠ private 후보
.env가 commit됨🔒 즉시 private + git history 정리

함정

  • 너무 일찍 archive: 1-2개월 안 만진 거라고 archive하면 다시 작업 시 활성화 필요. 6개월 임계.
  • README 0 repos: GitHub은 description 있어도 README 없으면 "incomplete" 인상. archive 전에 짧은 README라도.
  • license 누락: license 없는 repo는 외부 사용자 입장에서 "사용 불가". MIT 한 줄이라도.
  • archive해도 노출: GitHub 검색·프로필에서 여전히 보임. 완전 숨기려면 private.
  • delete 후 못 돌림: GitHub repo delete는 일정 기간 안에 복원 가능하지만 보통 즉시 영구. archive·private이 안전.
  • 모든 학습 레포 archive: "학습 멈춤"으로 보일 수 있음. 진행 중인 한두 개는 active 상태 유지.

핵심

6년 누적 학습 레포는 자산이자 노이즈다. Keep/Archive/Private/Delete 4분류로 분기마다 정리하면 프로필이 "현재의 본인"을 정확히 보여준다.

    GitHub Pages(사이트) + Releases(CDN) + Notion(DB) + Actions(cron). 신용카드 0원으로 운영되는 personal stack. side project 시작 비용을 0으로 만든다.

  • 🌿 spike — 미지의 영역은 버릴 코드로 학습한다

    모르는 기술/API/문제는 바로 v1.0을 만들 수 없다. 30분~2시간짜리 spike로 한 가지만 배우고 버린다. 본 구현은 그 학습 위에 처음부터 다시.
