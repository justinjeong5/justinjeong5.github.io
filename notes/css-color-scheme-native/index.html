CSS color-scheme — 다크 모드에서 form·scrollbar가 자동 따라오게

`color-scheme: dark` 한 줄로 form input·scrollbar·browser native UI가 OS 다크 모드에 자동 적응. 직접 스타일링 부담 ↓.

  • #css
  • #dark-mode
  • #accessibility
  • #frontend

문제

CSS 다크 테마를 직접 만들면 보통 background·color만 바꾼다. 그러나 다음은 OS·브라우저 native:

  • <input>, <select>, <textarea> 기본 스타일
  • 스크롤바
  • 자동완성 색
  • 텍스트 선택 색

이게 light 톤 그대로면 다크 모드에서 흰 input이 박혀 어색.

해법

:root[data-theme='dark'] {
  color-scheme: dark;
}

이 한 줄로 위 모든 native UI가 다크 모드로 자동 전환.

동작 원리

color-scheme 속성은 브라우저에게 "이 페이지가 어떤 스킴을 지원하는지" 알린다:

  • light — 라이트 UI 사용
  • dark — 다크 UI 사용
  • light dark — 둘 다 지원, 사용자 OS 설정 따름

브라우저는 이 힌트로 native form·scrollbar 등의 색을 자동 결정.

추가 활용

:root {
  color-scheme: light dark;  /* OS 설정 자동 따름 */
}

input { accent-color: var(--accent-red); }  /* checkbox·radio 색 */

accent-color까지 합치면 form 인풋 80% 다크 모드 대응 끝.

함정

  • 일부 브라우저(특히 구형 Safari)는 color-scheme 일부만 지원
  • meta name="theme-color"와 짝으로 써야 iOS Safari status bar까지 적응
  • 너무 강한 차이의 다크 색이면 native UI가 어색할 수 있음 — 중간 톤 권장

핵심

다크 테마는 background·color 토큰만으로는 부족하다. native UI까지 다크 모드를 알려야 진짜 완성.
