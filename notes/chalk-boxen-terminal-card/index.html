chalk + boxen으로 터미널 시각 카드 — ANSI fallback까지

터미널은 평면 텍스트가 아니다. chalk(색)·boxen(박스)·figlet(ASCII art)로 잡지(magazine) 같은 카드. NO_COLOR·width 변동·이모지 깨짐 대비.

  • #cli
  • #npm
  • #terminal
  • #design

문제 — 터미널 출력은 보통 plain하다

CLI 도구를 만들 때 출력은 보통 console.log 한 줄. 시각적 위계 없음.

좋은 CLI는 시각적 위계가 있다.

  • 제목·강조는 색·굵게
  • 그룹은 박스로
  • 상태는 색·이모지

3가지 라이브러리 조합으로 충분.

chalk — 색·스타일

import chalk from 'chalk';

console.log(chalk.bold.cyan('Title'));
console.log(chalk.dim('subtitle'));
console.log(chalk.green('✓ success'));
console.log(chalk.red('✗ error'));

색 종류 16개 기본, RGB 256색, true color. 환경 감지 자동.

boxen — 박스 박스

import boxen from 'boxen';

console.log(boxen('Hello\nWorld', {
  padding: 1,
  margin: 1,
  borderStyle: 'round',
  borderColor: 'cyan',
  align: 'center',
}));

borderStyle: single·double·round·bold·classic 등. round가 가장 부드러움.

조합 — 명함 카드 예시

import chalk from 'chalk';
import boxen from 'boxen';

const card = [
  chalk.bold.white('Justin Jeong') + chalk.dim(' / Senior FE'),
  '',
  chalk.cyan('github')  + '  github.com/justinjeong5',
  chalk.cyan('email')   + '   justin@example.com',
  chalk.cyan('site')    + '    justinjeong5.github.io',
].join('\n');

console.log(boxen(card, {
  padding: 1,
  margin: 1,
  borderStyle: 'round',
  borderColor: 'cyan',
}));

30줄도 안 되는 코드가 시각적 카드가 된다.

ANSI fallback (NO_COLOR·CI)

NO_COLOR 환경변수가 있거나 process.stdout.isTTY === false (pipe·redirect)면 색을 꺼야 한다.

chalk은 자동 감지. 다만 boxen은 박스 문자 자체는 unicode라 일부 환경에서 깨짐.

const supportsUnicode = process.env.TERM_PROGRAM === 'vscode' || process.platform !== 'win32';

const borderStyle = supportsUnicode ? 'round' : 'classic';

classic은 ASCII (+, -, |) 기반이라 어디서나 동작.

함정

  • chalk v5+ ESM 전용: CommonJS에서 require('chalk') 깨짐. v4까지가 CJS 호환. ESM 적용 안 됐으면 chalk@4.
  • width 가정 hardcode: 터미널 너비 80자 가정하면 좁은 창에서 줄바꿈 깨짐. process.stdout.columns 감지.
  • 이모지 환경 의존: macOS·Linux는 OK, Windows cmd·일부 SSH는 깨짐. fallback char 준비.
  • NO_COLOR 무시: 색 강제하면 접근성 떨어짐. chalk default 따르기.
  • boxen padding 0: 텍스트가 박스 모서리에 붙으면 답답. padding 1+ 권장.
  • 너무 많은 색: 5색 이상은 시각적 노이즈. 1-2색 강조 + 나머지 dim.

핵심

터미널 출력에 위계만 줘도 사용자 신뢰가 비선형으로 올라간다. chalk + boxen 두 라이브러리면 충분하다. NO_COLOR·width·이모지 fallback이 production 차이.

  • 🌿 한국어 letter-spacing — Pretendard에서 -0.025em이 상한

    영문 헤딩의 -0.04em letter-spacing은 한국어에 적용하면 받침 있는 자모가 뭉친다. -0.02em ~ -0.025em이 한글 균형점.

  • 🌿 npm 패키지 출시 체크리스트 — bin·files·engines·dry-run

    npm publish 한 번이 영구다. bin 경로·files glob·engines·--dry-run·SemVer 6가지를 사전 검증. 실수 한 줄이 100명의 install error를 만든다.

  • 🌿 npx <name> — SWE 첫 인상의 새 형태

    명함을 npm 패키지로. `npx justinjeong` 한 줄이 GitHub 링크보다 강한 인상을 준다. 터미널이 만남의 자리인 SWE 사이에서.

    모호한 품질 목표를 측정 가능한 rubric으로 바꾸고, 페르소나 평가로 채점한 뒤, 통과까지 개선·재평가하는 패턴.
