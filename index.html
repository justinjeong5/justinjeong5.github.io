자라고 있는 노트

완성된 글이 아니라 토픽별로 자라는 노트들입니다. 🌱 → 🌿 → 🌳.

  • 🌳

    Codex 경쟁 — 모든 작업을 두 번 굴려보기

    코딩·추론·리뷰 등 모든 작업에서 Codex와 Claude를 병렬 실행하고 결과를 교차 검증. 토큰 비용은 늘지만 사각지대가 줄어든다.

  • 🌱

    GitHub Pages에서 SPA 라우팅을 동작시키는 404 트릭

    GitHub Pages는 SSR이 없어 클라이언트 라우팅 새로고침 시 404가 뜬다. public/404.html에 SPA fallback redirect를 두면 해결.

  • 🌳

    좋은 알람의 4가지 기준 — 노이즈와 신호의 분리

    알람이 많은 시스템은 알람이 없는 시스템과 같다. 좋은 알람은 사람이 지금 행동해야 할 때만 울린다. 4가지 기준으로 노이즈를 걷어내는 법.

  • 🌳

    Incrementally correct — 한 번에 완성하지 않는 사이트

    Brian Lovin이 제안한 개념. 글이 "출간"되는 게 아니라 시간을 두고 점진적으로 정확해지는 운영 방식.

  • 🌳

    콘텐츠 = 파일 — MDX + Vite의 자동 확장 패턴

    콘텐츠 한 개 = MDX 파일 한 개. import.meta.glob으로 자동 인덱싱하면 코드 수정 없이 사이트가 확장된다.
