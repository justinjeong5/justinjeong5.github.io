모바일 최적화 — iOS HIG·viewport·접근성을 한 사이클에

iPhone 15(393px)에서 GNB가 viewport 초과한 단편 이슈로 시작. 3 페르소나 × 8축 quality-gate를 3 사이클 돌려 평균 2.75 → 4.75 통과. iOS HIG 44×44, viewport-fit, 100dvh 등 종합.

Role
디자인·반응형 구현·접근성·iOS Safari 디테일
Period
2026.05
Tags
#mobile #responsive #ios #accessibility

문제

데스크탑에서 디자인 통과 후 사용자가 보고:

"아이폰 15로 접속하니까 상단 GNB가 화면 너비를 넘어간다."

GNB 한 영역만 단편 수정으로 끝낼 수 있었지만, 그러면 다른 모바일 이슈가 누적될 거였다. 종합 점검이 필요했다.

의사결정

버린 선택지

  • GNB만 햄버거 메뉴로 수정 후 종료: 다른 영역 잠재 이슈 누락
  • 모바일 전면 리디자인: 본질 문제는 GNB·터치·접근성 등 디테일 누락

채택

  • 3 페르소나 (디자이너·기획자·개발자) × 8축 rubric quality-gate
  • 평균 4.5/5 + 모든 항목 ≥ 4 통과까지 사이클

사이클 결과

평균 2.75 → 4.75/5 PASS (모든 항목 ≥4)

산출물

iOS Safari 패키지

  • viewport-fit=cover + theme-color light/dark meta
  • safe-area-inset env() (.site-nav top, .site-footer bottom)
  • 100vh → 100dvh 일괄 (body, main, modal-card)
  • -webkit-tap-highlight-color: transparent 글로벌
  • -webkit-backdrop-filter 폴백
  • -webkit-text-size-adjust: 100%

터치 타겟 HIG 44×44

  • .icon-button 38 → 44, .search-trigger 38 → 44, .brand 42 → 44, .button 42 → 44
  • .see-all-link 모바일 min-height 44

인터랙션·접근성

  • CommandPalette input mobile attrs (inputMode·autoCorrect·autoCapitalize·enterKeyHint)
  • 3 모달에 previouslyFocused 캐시 + 닫힘 시 trigger 복귀
  • inert 패턴 (focus trap 대체) — modal 열림 시 background subtree 비활성화
  • :focus-visible 글로벌 outline + prefers-reduced-motion 가드
  • @media (hover: none) 카드 sticky hover 방지

콘텐츠 우선순위

  • 모바일에서 .profile-panel { display: none } (currentFocus·heroProof 중복 해소)
  • hero padding 56/48 축소, hero-actions row + flex-wrap
  • LogsPage 본문 <details><summary> 토글

임팩트

  • iPhone 15 / SE 375 / Pro Max 430 모두 viewport overflow 0건
  • 헤더 핵심 액션 3개 (검색·테마·메뉴) 모두 HIG 44×44 준수 → 미스 터치 감소
  • 모바일 첫 viewport에 hero copy + CTA + visitor paths 도달
  • WCAG 2.2 1.4.13/2.4.7/2.4.11 + APG dialog 패턴 충족

회고

평균이 아니라 최저 항목이 통과를 결정. iOS Safari 특수성 2점이 핵심 BLOCKING이었음. viewport-fit·safe-area·100dvh 같은 작은 메타데이터가 모바일 신뢰감의 80%.
