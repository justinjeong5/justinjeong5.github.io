About

정경하

제품 문제를 빠르게 구조화하고, 프론트엔드를 축으로 백엔드·자동화까지 연결해 실제 운영 흐름을 개선하는 엔지니어입니다. 모호한 요청을 실행 단위로 쪼개고, 다수의 AI 에이전트와 자동화로 검증 가능한 속도를 만들며, 그 판단과 결과를 공개 사례로 남깁니다.

Evidence system — 케이스가 강해지는 네 칸

01

Problem

누가 어떤 상황에서 막혔는지 한 문장으로 적습니다.

사용자 흐름, 운영 로그, 요청 원문
02

Decision

선택지와 버린 이유를 남겨 판단력을 보여줍니다.

옵션 비교, trade-off, 범위 결정
03

Output

실제 산출물을 볼 수 있게 링크·캡처를 붙입니다.

PR, 데모, 스크린샷, 문서
04

Impact

수치가 있으면 수치로, 없으면 관찰 가능한 변화로 적습니다.

시간 절감, 오류 감소, 응답 증가, 회고

Operating principles

  • 문제를 먼저 고정합니다

    화면·API·자동화보다 먼저 누가 어떤 불편을 겪는지와 성공 조건을 문장으로 고정합니다.

  • 증거가 남는 방식으로 만듭니다

    배포 링크, 로그, 스크린샷, 테스트, 전후 비교 중 최소 하나가 남도록 작업을 설계합니다.

  • 작게 배포하고 다시 읽습니다

    한 번에 완성하려 하기보다 작은 릴리즈 후 사용자 반응과 운영 비용을 다시 확인합니다.

  • 다음 사람이 고치기 쉽게 둡니다

    콘텐츠와 화면 구조를 분리하고, 반복되는 사례는 데이터만 추가해 확장되게 만듭니다.

대표 작업

  • 홈 레이더 — 부동산 자동 수집

    주 1–2시간 수작업을 0분으로. 공개 API 7곳 + 노션 백엔드 + GitHub Actions cron.

  • 온보딩 문서 시스템

    첫 PR을 5–7일에서 2–3일로, 첫 배포 책임을 3개월에서 1.5개월로.

  • dispatch 라우팅 — AI 워크플로우

    3 에이전트 경쟁 + 점수 공식으로 의도 기반 스킬 라우팅을 자동화.

  • 양면 시장 의사결정 — 광고 플랫폼

    광고주와 매체의 인센티브가 충돌하는 지점에서, 기본값과 모니터링을 약한 쪽으로 설계한 판단.
