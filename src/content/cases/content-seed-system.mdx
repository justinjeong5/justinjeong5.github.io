## 문제

라우팅·MDX·디자인·모바일 인프라가 완성됐지만 콘텐츠가 시드 3개뿐. 채용 담당자가 보면 "빈 액자" 인상. 사이트가 살아있다는 신호 부재.

문제는 시간. 본업 8시간 외에 매일 글 쓸 부담은 sustainable 하지 않다. 그렇다고 빈 사이트는 가치 없음.

## 의사결정

### 버린 선택지
- **자동 생성으로 11편 만들기**: AI 톤 오염, 1인칭 일관성 ↓
- **본인 의견 전부 직접 작성**: 시간 부담, 한 세션 완료 불가
- **사이트 publish 미루기**: 인프라가 죽은 상태로 누적

### 채택

**책임 분리**:
- **Claude가 1차 작성**: 메타 작업·도구·이번 세션 사실 기반 (객관 가능 영역)
- **사용자가 검토·다듬기**: 1인칭 톤·뉘앙스·결정 이유 (주관 영역)
- **quality-gate 통과 확인**: 객관 기준으로 ship 여부 결정

## 임팩트

- 빈 사이트 → 11편 콘텐츠로 시각적 가든 살아남
- frontmatter 스키마 표준화 — 향후 콘텐츠 추가가 데이터 기입만으로 자동 노출
- 메타 시스템 노출 — Claude Code·dispatch·quality-gate 시스템을 글로 외화

## 회고

빈 인프라는 인프라가 아무리 좋아도 죽은 액자. 핵심 인사이트: **Claude가 잘 하는 영역과 사용자만 할 수 있는 영역 분리**. 메타 작업·이번 세션 사실 = Claude. 본인 의견·뉘앙스 = 사용자.

이 분리가 작동하니 한 세션에 11편 추가가 sustainable.
