## 문제

리빌드 직후 사이트 톤이 6년 전 brutalist 잔재 (검은 띠, 큰 빨간 step 번호, 그림자 노이즈)로 채워져 있었다. 사용자 1차 피드백: **"엄청 못생겼어"**.

직관 수정으로는 어디부터 손대야 할지 불명확. 평가 시스템이 필요했다.

## 의사결정

### 버린 선택지
- 색상 미세 조정만 — 토큰 부재 상태에서 hex 30+ 곳 일관성 불가
- 전면 리디자인 (3D / Gamified) — 미니멀 톤·운영 비용과 충돌
- 부분 패치 — 동일 함정 반복

### 채택
- **디자인 토큰 :root variables로 추출 → 단일 source 회복**
- Pretendard Variable (dynamic-subset) 도입
- 4 페르소나 quality-gate 사이클로 점수 ≥ 4.5 통과까지

## 산출물

### Phase 1: Editorial Modern (사이클 1)
- Pretendard Variable dynamic-subset (풀 2MB → 청크 ~40KB×N)
- Light: cream `#faf9f5` + charcoal `#18181b` + burnt-orange `#c2410c`
- Dark: almost-black + cream
- font-weight 950/900 → 600/700 다이어트
- 카드 box-shadow 제거 → hairline border, 모서리 8px → 6px

### Phase 2: 누런 톤 박멸 (사이클 2·3)
- cream → **pure white `#ffffff`** + zinc + indigo `#4f46e5`
- 하드코딩 hex 30+ 곳, rgba 12+ 곳 → `var(--*)` 일괄 치환
- `--text-inverse: #fafafa` 토큰 신설 (다크 배경 위 light text 의미)
- sepia 테마 완전 삭제 (누런 톤 가장 강함)
- HomePage `heroProof` 3카드 활성화 (dead asset)

## quality-gate 결과

| 항목 | C1 | C3 |
|---|---|---|
| 컬러 팔레트 (BLOCKING) | 2 | **5** |
| 타이포그래피 (BLOCKING) | 3 | **5** |
| 시각 일관성 | 2 | **5** |
| 다크 테마 | 3.5 | **5** |
| 첫인상·임팩트 | 3 | 4 |

**평균 3.28 → 4.44/5** (3 사이클)

## 임팩트

- "엄청 못생겼어" → "좋아 마음에 들어"
- hex/rgba 잔재 0건 grep 확인 (single source of truth)
- 한국어 letter-spacing -0.025em으로 받침 자모 뭉침 해소
- 다크 모드 form/scrollbar `color-scheme: dark` 한 줄로 자동 적응

## 회고

세련된 디자인은 **토큰 일관성이 80%**. 사용자가 hex 잔재를 의식적으로 못 보지만 시각적 안정감의 무게가 다르다. 그리고 사용자가 누런 톤을 싫어한다 → burnt-orange도 같은 계열로 느낀다는 신호. 페르소나 평가가 짚지 못했다면 못 잡았을 디테일.
