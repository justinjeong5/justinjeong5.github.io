## 44×44는 왜 필요한가

Apple Human Interface Guidelines가 "최소 44pt"를 권장하지만, 실제로는 더 강한 의미가 있다. 사람의 손가락 패드는 평균 10mm. 44pt는 약 11mm. 그보다 작으면 미스 터치가 누적된다.

이 사이트에서 `.icon-button`을 38×38에서 44×44로 바꾸자 헤더의 검색·테마·메뉴 트리거 누르기가 눈에 띄게 안정됐다.

## 어디까지 적용해야 하는가

- ❗ 모든 인터랙티브 요소 (button, link, input, select)
- ❗ 카드 자체가 큰 클릭 영역이면 카드는 OK. 단 카드 안의 작은 link는 별도 처리.
- 시각 사이즈는 작아도 padding으로 hit area 확보 가능 (`padding: 10px 0` + `min-height: 44px`)

## safe-area-inset

iPhone 11+는 노치, iPhone 14 Pro+는 Dynamic Island. 하단은 홈 인디케이터. 콘텐츠가 그 영역에 닿으면 잘리거나 가려진다.

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, viewport-fit=cover" />
```

```css
.site-nav {
  padding-top: max(14px, env(safe-area-inset-top));
}
.site-footer {
  padding-bottom: max(24px, env(safe-area-inset-bottom));
}
```

`max()`로 일반 padding과 inset 중 큰 값. 노치 없는 폰에선 일반 padding, 있는 폰에선 inset.

## 100dvh

iOS Safari는 주소창이 동적으로 사라졌다 나타난다. `100vh`는 주소창 펼친 상태 기준이라 사라지면 콘텐츠 점프.

`100dvh`(dynamic viewport height)는 현재 viewport 기준이라 점프 없음. iOS 15.4+ / Chrome 108+ 지원.

## 작은 디테일 묶음

- `-webkit-tap-highlight-color: transparent` — 링크 탭 시 회색 깜빡임 제거
- `-webkit-text-size-adjust: 100%` — 가로 회전 시 폰트 자동 확대 방지
- `input { font-size: 16px }` — iOS Safari 자동 줌 트리거 회피
- `overscroll-behavior: contain` — 모달 스크롤 끝에서 백드롭이 함께 스크롤되는 것 차단

## 함정

- **44×44를 한 군데 빼먹기**: 사용자는 "왜 이 버튼만 누르기 어렵지?"라고 느끼지만 원인을 정확히 못 짚는다. 한 곳 누락이 전체 인상을 깎는다.
- **시각 사이즈만 줄이고 hit area 안 늘림**: 작은 아이콘은 padding으로 hit area를 따로 확보해야. 시각 12px / hit 44px 분리.
- **safe-area-inset을 max 없이 사용**: 노치 없는 폰에서 inset이 0이면 padding 0 → 너무 가까운 가장자리. 항상 `max(default, env(...))`.
- **dvh 미지원 브라우저 무시**: 100dvh fallback 없으면 옛 iOS에서 점프. `min-height: 100vh; min-height: 100dvh;` 둘 다.
