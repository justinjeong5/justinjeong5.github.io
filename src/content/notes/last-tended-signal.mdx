## 일반 블로그의 함정

블로그 글에는 publishedAt이 있다. 시간이 지나면 "오래된 글"로 보인다. 2년 전 글은 자동으로 신뢰 ↓.

문제: 글의 **내용**은 계속 다듬어지는데도, 사이트는 "2023-01-15"라는 한 시점에 박혀 있다.

## 디지털 가든의 해법

`plantedAt` (처음 심은 날) + `lastTendedAt` (최근 다듬은 날). 둘 다 표시.

```yaml
plantedAt: 2026-04-12  # 처음 심음
lastTendedAt: 2026-05-19  # 어제 다듬음
```

방문자가 보는 것:
- "2026-04-12 심음, 2026-05-19 마지막 다듬음"
- → "이 노트는 한 달간 계속 자라고 있다"
- → "이 사이트는 살아있다"

## 정렬 효과

`lastTendedAt`을 sort key로 쓰면, **최근에 다듬어진 노트가 상단**. 게시일 기준 정렬은 "1년 전 글이 영원히 1년 전 글"이지만, 다듬은 날 기준은 "현재 관심사가 자연스럽게 상단".

## 함정

- 매번 lastTendedAt 수동 업데이트 = 부담. git commit timestamp로 자동화 가능.
- "다듬었다"의 기준 모호 — 오타 수정도 카운트? 본문 5문장 추가는?
- 너무 자주 lastTendedAt 갱신 = 노이즈. 의미 있는 다듬기에만.

## 핵심

> 사이트의 "살아있음"은 게시 빈도가 아니라 다듬기 빈도다.
